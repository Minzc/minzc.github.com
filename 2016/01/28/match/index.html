Mining Long Sequential Patterns in a Noisy Environment

Posted on 2016-01-28   |  

Mining Long Sequential Patterns in a Noisy
Sigmond 2002
Jiong Yang, Jiawei Han, etc.

Abstract

  • Match Model
  • Estimate match value on a sample using Chernoff bound
  • Border collapsing to reduce the number of database scaning

    Motivation

    Long patterns have many applications, especially in gene analysis. The common metric used to qualify a significant pattern is support. Due the presence of noise, a symbol may be mispresented as another one. As a result, a frequent may be concealed by noise. To tackle this problem, we need a more flexiable model.

    Match

    Compatible Matrix
    Elements in the matrix shows the probability of the observe value being a real underlying substance.

Given a pattern $P=d_{1}d_{2}…d_{n}$ and sequence $S=m_{1}m_{2}…m_{n}$. The match of $P$ in $S$ is denoted as
$$M(P,S)=\Pi P(d_{i}|m_{i})$$.

If $S$ is longer than $P$, the match between $S$ and $P$ is defined as the maximal match of $P$ in every distinct subsequence with the lenght of $l_{p}$ of $S$.

Given a pattern $P$ and sequqnce database $D$, the match of $P$ is $D$ is defined as
$$M(P,D)=\frac{\sum M(P,S)}{N}$$

The match has Apriori property. The match of $P$ in database is $\leq$ the match of ang subsequence of $P$ in the database

Sampling

First, draw a sample from the database and use existing methods to mine the sample. Given the match of patterns in the sample data, we can use Chernoff bound to estimate the match of those patterns in the whole database.

All patterns will be labeled as Frequent, Infrequent or Ambiguois. We only need to calcualte the actue match score of Ambiguois patterns.

Border collapsing

Ambiguous patterns are embraced by infrequent patterns and frequent patterns. We want to collapse the two border and finally find the really frequent pattern border on the pattern lattice.

If memory can hold all of the patterns, only one database scan is fine. Sometimes, the number of ambiguiout patterns is very large. In this case, we select a set of ambiguious patterns, scan the database and collapse the border.

Patterns with high collapsing power are selected first. We prefer patterns lying on the halfway of the lattice. If those patterns are frequent, then all of their subpatterns are frequent. While, if those patterns are infrequent, then all of their superpatterns are infrequent.

In each iteration, we first select pattern in the half way and then quarter way, etc. until the memory is filled up. A scan of the database is performed to compute the matches of those patterns. Finally, we use the result to collapse the border. If the memory can hold all patterns up to x layers, the size of the ambiguous patterns can be narrowed down the $1/x$ of the original one. As a result, if it takes $y$ scans before, now only $log_{x}{y}$ scans are required.
